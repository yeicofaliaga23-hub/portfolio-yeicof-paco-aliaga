# 🤖 Reconstrução de Aplicativo com IA
> Projeto realizado no Google AI Studio utilizando o modelo Gemini como assistente de desenvolvimento Full-Stack.

---

## 🔍 Etapa 1 – Análise da Interface
- Acesso ao webapp de referência fornecido.
- Observação detalhada dos **componentes visuais**: botões, menus, campos de entrada, áreas de exibição de dados.
- Mapeamento das **regras de negócio** percebidas:
  - Como os dados eram processados e exibidos.
  - Quais interações disparavam mudanças na interface.
  - Fluxo esperado do usuário em cada funcionalidade.

---

## ⚙️ Etapa 2 – Configuração no Google AI Studio
- Definição das **System Instructions** para o modelo Gemini:
  - Atuar como **desenvolvedor Full-Stack**.
  - Estrutura de arquivos necessária:
    - `index.html` → estrutura principal da interface.
    - `style.css` → estilização visual.
    - `App.js` → lógica em React para componentes dinâmicos.
    - `api.js` → simulação de chamadas de backend.
- Especificação dos **comportamentos esperados**:
  - Resposta imediata às interações do usuário.
  - Validação de entradas.
  - Atualização dinâmica da interface.
  - Estética fiel ao aplicativo original.

---

## 🛠️ Etapa 3 – Construção e Validação
- Geração do código completo (HTML, CSS, React e JS) pelo modelo.
- Execução do aplicativo no ambiente de teste do Google AI Studio.
- Comparação entre o app reconstruído e o original:
  - **Nome e identidade visual** mantidos.
  - **Comportamento funcional equivalente**.
  - **Estética fiel** à interface original.
- Iterações realizadas:
  - Testar o código gerado.
  - Identificar diferenças em relação ao app original.
  - Refinar instruções.
  - Validar novamente até alcançar o resultado esperado.

## ✅ Resultados Obtidos
- Aplicativo final reproduziu com sucesso o **nome, estética e comportamento** do webapp original.
- Demonstração da capacidade de usar IA como **parceira no desenvolvimento Full-Stack**, sem acesso ao código-fonte.
- Competências desenvolvidas:
  - **Análise de interfaces externas**.
  - **Mapeamento de lógica de negócio**.
  - **Configuração de instruções para IA**.
  - **Validação iterativa de software**.

---

## 📌 Conclusão
Este projeto demonstrou que é possível reconstruir um aplicativo funcional apenas pela observação da interface externa, utilizando a inteligência artificial como parceira estratégica no desenvolvimento.

A experiência reforçou não apenas a capacidade técnica de configurar instruções precisas e validar código gerado por IA, mas também a importância da análise crítica de interfaces, do mapeamento de regras de negócio e da validação iterativa como etapas fundamentais de engenharia de software.

O resultado final foi um aplicativo que preserva a identidade visual, o comportamento funcional e a experiência de usuário do sistema original, provando que a colaboração entre desenvolvedor e IA pode acelerar processos, reduzir barreiras técnicas e abrir caminho para soluções criativas e inovadoras.
